# 多电脑同步部署说明

目标：

```text
你这台电脑负责开发和优化代码。
对方电脑负责日常运行系统。
你优化后推送代码，对方执行一键更新脚本即可更新程序。
对方电脑上的库存数据、上传图纸、密钥配置不会被覆盖。
```

## 一、哪些文件会同步

会同步的内容：

```text
app/
miniprogram/
scripts/
requirements.txt
README.md
```

不会同步的本机数据：

```text
.env
.venv/
data/
backups/
```

这些已经写在 `.gitignore` 中。请不要把 `.env`、`data/app.db`、`data/uploads/` 提交到 Git。

## 二、你这台电脑优化后怎么发布

在你这台电脑进入项目目录：

```bash
cd /Users/luck/Desktop/杭州特耐时/backend
```

确认改了哪些文件：

```bash
git status
```

提交并推送：

```bash
git add .
git commit -m "优化程序"
git push
```

## 三、对方电脑第一次部署

先安装：

```text
Git
Python 3
微信开发者工具
```

然后下载项目。因为当前远程仓库名是 `-`，建议克隆时直接指定本地文件夹名：

```bash
git clone https://github.com/ly-825/-.git 杭州特耐时-backend
cd 杭州特耐时-backend
```

创建环境并安装依赖：

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt
```

Windows 使用：

```bat
python -m venv .venv
.venv\Scripts\python -m pip install -r requirements.txt
```

准备配置文件：

```bash
cp .env.example .env
```

如果需要千问识别，在 `.env` 中填写：

```text
DASHSCOPE_API_KEY=你的APIKey
```

如果是从旧电脑迁移业务数据，把旧电脑的这些内容复制到对方电脑：

```text
data/app.db
data/uploads/
```

## 四、对方电脑启动服务

macOS/Linux：

```bash
.venv/bin/python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Windows：

```bat
.venv\Scripts\python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

测试地址：

```text
http://127.0.0.1:8000/admin
http://127.0.0.1:8000/health
```

## 五、对方电脑以后怎么更新

进入项目目录：

```bash
cd 项目目录
```

执行：

```bash
bash scripts/update.sh
```

脚本会自动做这些事：

```text
1. 检查 Git 和 Python
2. 检查本地代码有没有未保存改动
3. 备份 data/app.db 和 data/uploads/
4. 拉取你推送的新代码
5. 安装或更新 Python 依赖
6. 提示重启后端服务
```

更新后需要停止旧服务，再重新启动后端。

## 六、小程序地址

对方电脑如果局域网 IP 变了，需要修改：

```text
miniprogram/app.js
```

把 `baseUrl` 改成对方电脑的新地址，例如：

```js
baseUrl: 'http://192.168.31.99:8000'
```

手机真机预览时，需要：

```text
1. 手机和电脑在同一个 Wi-Fi
2. 后端使用 --host 0.0.0.0 启动
3. 微信开发者工具勾选“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”
```

## 七、更新前后检查

更新前建议：

```bash
bash scripts/backup.sh
```

更新后检查：

```text
1. /health 返回 {"status":"ok"}
2. /admin 可以打开
3. 小程序工作台可以打开
4. 原有库存、图纸、余料数据还在
```
